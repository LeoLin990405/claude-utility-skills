# Claude 模型概览

## 1. Claude Opus 4.5 - 旗舰智能模型

### 核心定位
- Anthropic 最智能的模型
- 混合推理模型 (Hybrid reasoning)
- 200K 上下文窗口

### 性能指标
| 基准测试 | 得分 | 说明 |
|----------|------|------|
| SWE-bench Verified | 80.9% | 行业领先编程能力 |
| OSWorld | 66.3% | 计算机使用能力 |
| SWE-bench Multilingual | 7/8 语言最佳 | 多语言编程 |

### 定价
- 输入: $5 / 百万 tokens
- 输出: $25 / 百万 tokens
- 提示缓存: 最高节省 90%
- 批处理: 节省 50%

### 独特功能
1. **Effort 参数**: 控制响应详细程度
   - High: 最详尽分析
   - Medium: 平衡模式
   - Low: 最高效简洁

2. **Thinking Block 保留**: 自动保留所有推理历史

3. **Zoom 功能**: 计算机使用时可放大检查细节

### 适用场景
- 复杂推理任务
- 多步骤编码项目
- 企业级工作流
- 深度研究分析

## 2. Claude Sonnet 4.5 - 最佳编码模型

### 核心定位
- 最佳编码和代理模型
- 混合推理支持
- 200K / 1M (beta) 上下文窗口

### 性能特点
- SWE-bench Verified: 世界领先
- OSWorld: 61.4% (vs Sonnet 4 的 42.2%)
- 可持续专注 30+ 小时处理复杂任务

### 定价
- 输入: $3 / 百万 tokens
- 输出: $15 / 百万 tokens

### 代理能力
1. **长时间自主运行**: 数小时独立工作
2. **上下文感知**: 实时追踪 token 使用
3. **并行工具调用**: 同时执行多个搜索/读取
4. **状态追踪**: 跨会话保持目标导向

### 适用场景
- 软件开发全生命周期
- 长期运行的 AI 代理
- 浏览器/计算机自动化
- 金融分析
- 网络安全

---

## 3. Claude Haiku 4.5 - 最快速模型

### 核心定位
- 最快、最具成本效益
- 近前沿性能 (匹配 Sonnet 4)
- 200K 上下文窗口

### 性能指标
- SWE-bench Verified: 73.3%
- 速度: Sonnet 4 的 2 倍以上
- 成本: Sonnet 4 的 1/3

### 定价
- 输入: $1 / 百万 tokens
- 输出: $5 / 百万 tokens

### 新增能力 (首次)
1. **扩展思考**: 首个支持 extended thinking 的 Haiku
2. **上下文感知**: 实时追踪剩余上下文
3. **完整工具支持**: bash、代码执行、文本编辑、网页搜索、计算机使用

### 适用场景
- 实时交互应用
- 高并发处理
- 免费层 AI 服务
- 多代理系统子代理
- 大规模计算机自动化
